# reports/

Auto-generated analyses and metrics from the dataset pipeline. These are
regenerated by the scripts in [`scripts/`](../scripts/) and are tracked so the
project's data decisions are auditable and reproducible.

| File                              | Produced by                          | Description                                            |
|-----------------------------------|--------------------------------------|--------------------------------------------------------|
| `dataset_report.md`               | `analyze_datasets.py`                | Per-source inventory, label health, duplicates.        |
| `class_distribution.md` (+ CSVs)  | `class_distribution.py`              | Instances and images per class, per source.            |
| `merged_class_distribution.md`    | `merged_class_distribution.py`       | Class distribution of the final merged dataset.        |
| `final_statistics.md`             | `generate_final_stats.py`            | Final image/instance counts per split.                 |
| `leakage_report.md`               | `check_leakage.py`                   | Exact + near-duplicate cross-split leakage audit.      |
| `russian_military_invalid_boxes.md` | `report_russian_invalid_boxes.py`  | Analysis of invalid boxes removed during cleaning.     |
| `final_report.pdf`                | *(manual)*                           | Written project report (add your PDF here).            |

### Regenerate

```bash
python scripts/run_prepare_pipeline.py   # runs cleaning, merge, stats end-to-end
python scripts/check_leakage.py          # leakage audit
```

> `final_report.pdf` is the only manually authored artifact in this folder; place
> your compiled report PDF here so it ships with the repository.
