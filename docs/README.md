# docs/

Visual documentation and figures referenced by the main [`README.md`](../README.md).

Place the following assets here. These are **figures**, not raw data, so they are
tracked by git (an exception is added in `.gitignore`).

| File                     | What it should show                                                        |
|--------------------------|----------------------------------------------------------------------------|
| `results.png`            | Training/validation curves or the model-comparison summary for YOLO11n/s/m. |
| `confusion_matrix.png`   | Confusion matrix of the final **YOLO11m** model on the test split.          |
| `sample_detection_1.jpg` | Example detection on real UAV footage (e.g. a tank / armored convoy).       |
| `sample_detection_2.jpg` | Second example detection (e.g. mixed military + civilian vehicles).         |

### How to generate

- `results.png` / `confusion_matrix.png` are produced by Ultralytics during
  validation under `runs/detect/<run>/`. Copy them here:

  ```bash
  yolo detect val data=data.yaml model=runs/detect/train/weights/best.pt
  cp runs/detect/val/confusion_matrix.png docs/confusion_matrix.png
  cp runs/detect/val/results.png          docs/results.png
  ```

- Sample detections can be generated by running prediction on a video frame:

  ```bash
  yolo detect predict model=models/best.pt source=videos/<clip>.mp4 save=True
  ```

> Keep figures lightweight (compress large PNGs/JPGs) so the repository stays clone-friendly.
